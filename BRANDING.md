# Haven 42 Brand And Compatibility

## Product Identity

- Product name: Haven 42
- Tagline: Your private, local AI station.
- Canonical repository: `hysel/haven-42`
- Release artifact prefix: `haven-42-<version>`
- Previous name: Local Engineering Agent Pack

Haven 42 describes the complete local-first product direction: software-engineering workflows, repository-optional general AI tasks, local provider capabilities, typed artifacts, evidence, and the planned local web experience.

## Clean-Rename Policy

The project had no external users when Haven 42 was selected. Product-specific identifiers therefore adopt the Haven 42 name directly instead of carrying an indefinite legacy compatibility layer. This includes:

- Guided-menu workflow IDs and script filenames.
- Shared-assets and managed MLX environment directories.
- Release artifact names and temporary session paths.
- Provider client identifiers, canonical links, and public documentation.

Generic engineering terms such as "agent pack" may still appear when they describe a component rather than the product. Historical changelog and evidence statements retain the name that was accurate when the event occurred.

## Repository Rename

GitHub redirects the previous repository and wiki URLs after the rename. Maintainers should still update local remotes to the canonical URLs. Do not create a new repository under the previous `local-engineering-agent-pack` name because doing so would disable GitHub's redirects.

The local checkout directory does not have to be renamed immediately. Close editors and running shells before renaming a checkout folder, then reopen the repository from its new location.
